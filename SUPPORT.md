# Support

Multi-Git is a community-maintained open-source project. Help is provided on a best-effort basis without guaranteed response or resolution times.

## Before Asking for Help

1. Read the installation, first-use, SSH, troubleshooting, and security sections in the [README](README.md).
2. Update to the [latest release](https://github.com/AnthonyKopri/multi-git/releases/latest).
3. Open the Terminal Log from the top toolbar and review the most recent command and error.
4. Confirm that Git, `ssh`, and `ssh-keygen` are available on `PATH`.
5. Search existing [issues](https://github.com/AnthonyKopri/multi-git/issues) for the same behavior.
6. Reproduce the problem with a disposable repository when possible.

## Where to Ask

- Use the bug report form for reproducible application defects.
- Use the feature request form for a new workflow or improvement.
- Add focused information to an existing issue when it describes the same problem.
- Follow [SECURITY.md](SECURITY.md) for suspected vulnerabilities.

Do not use public issues for private keys, passphrases, vault master keys, tokens, private repository contents, or vulnerability details.

## Helpful Diagnostic Information

When reporting a normal bug, include:

- Multi-Git version and installer type.
- Windows version or other operating system details.
- Git, Node.js, and OpenSSH versions when relevant.
- Whether the remote uses SSH or HTTPS.
- Whether the problem occurs with System SSH, a saved profile, or both.
- Minimal steps to reproduce the behavior.
- Redacted Terminal Log output.

Replace usernames, organization names, repository paths, remote URLs, emails, and other sensitive values when they are not needed to reproduce the issue.
